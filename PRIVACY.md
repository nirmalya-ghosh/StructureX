# StructureX Privacy Policy

Effective date: May 4, 2026

This Privacy Policy explains how StructureX handles information in the browser dashboard, static deployment, API routes, demo authentication flow, and related repository materials.

Do not upload personal, confidential, restricted, classified, or regulated infrastructure data unless you have authority to process it and understand the risk.

## Information Processed

StructureX may process:

- name and email entered in login or signup forms
- local browser session data, including mock user session state and dashboard activity history
- uploaded CSV files and scenario inputs
- map coordinates, search queries, selected building metadata, weather targets, and generated risk outputs
- browser, device, network, request, and hosting metadata
- technical logs created by local development tools, hosting providers, external APIs, or security systems

## How Information Is Used

Information is used to:

- authenticate the demo session and personalize the dashboard
- run building, scenario, resilience, satellite, and dataset analysis features
- operate, secure, debug, improve, and monitor the service
- prevent misuse, unsafe reliance, fraud, abuse, and unauthorized access
- maintain repository, deployment, and operational records

## Local Storage and Cookies

StructureX uses browser storage for session state and local dashboard activity. You can clear this by signing out, using in-app clear actions, or clearing browser site data.

Hosting providers and third-party services may use cookies or similar technologies for authentication, security, analytics, abuse prevention, and service delivery.

## Third-Party Services

StructureX may connect to services such as Vercel, MapTiler, Open-Meteo, Nominatim/OpenStreetMap, Google-hosted assets, Font Awesome, CDN providers, and AI/model providers. These services may receive request metadata such as IP address, user agent, referrer, coordinates, search terms, API paths, timing data, and error data.

Review third-party policies before using features that send data to external services.

## Sensitive Data Restrictions

Do not upload or enter:

- personal data, health data, financial data, credentials, or government identifiers
- classified, export-controlled, restricted, or confidential files
- non-public emergency plans or critical infrastructure vulnerability details
- data about private individuals or protected locations without authorization
- information that you are not legally permitted to process

You are responsible for redaction, consent, legal basis, retention rules, and compliance obligations for uploaded or entered data.

## Security

StructureX uses reasonable technical and organizational measures for a portfolio/demo deployment, but no internet service is perfectly secure. Do not use StructureX as the sole repository for critical records, safety decisions, or incident response.

## Retention

Browser-local data remains until cleared by the user or browser. Serverless request data is processed to return responses and may be subject to hosting provider logs, retention settings, and security monitoring.

Repository history may retain committed files indefinitely. Do not commit secrets, credentials, sensitive infrastructure data, or private datasets.

## Your Choices

You can:

- avoid uploading sensitive data
- clear browser storage and sign out
- fork or self-host the project under your own controls
- review third-party policies before using map, weather, CDN, or AI-backed features
- contact the maintainer through GitHub for privacy or security concerns

## Children

StructureX is not intended for children or for collecting information from children.

## Changes

This policy may be updated as StructureX changes. Continued use after updates means you acknowledge the revised policy.
